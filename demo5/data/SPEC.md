# 주간 데이터 생성 명세 (SPEC) — 반드시 그대로 지킬 것

이 문서는 12개 주차 데이터의 **구조를 100% 통일**하기 위한 계약이다.
컬럼명/순서/타입/인코딩을 하나라도 어기면 나중 분석 코드가 깨진다.

## 공통 규칙
- 인코딩: **UTF-8 with BOM** (`encoding="utf-8-sig"`), 구분자 `,`
- 각 파일 **행 수는 정확히 500행** (헤더 제외). 모든 고객이 매 파일에 1회 등장.
- 고객 순서: `customers.csv`의 `고객ID` 오름차순 그대로.
- 금액 컬럼은 **원(KRW) 단위 정수** (예: 83000000). 음수 금지(0 이상).
- `기준일자`: `calendar.csv`의 해당 주차 값(YYYY-MM-DD 문자열).
- `주차`: 정수 1~12.
- 결측치(NaN) 금지. 값이 없으면 0.

## 읽어야 할 입력 파일 (읽기 전용, 수정 금지)
- `data/customers.csv`      : 고객 마스터 (고객ID, 고객명, 연령, 성별, 지점코드, 지점명, 가입일자)
- `data/customers_base.csv` : 값 계산용 내부 파라미터
- `data/calendar.csv`       : 주차→기준일자, 폴더명

## 출력 파일 (자기 주차 폴더 `data/week_XX/` 에 4개)
컬럼명과 순서는 **정확히** 아래와 같아야 한다.

1. `여신잔액.csv`
   `고객ID,기준일자,주차,상품구분,여신잔액`
   - 상품구분 = `customers_base.csv`의 `여신_상품구분`
   - 여신잔액 = 아래 [값 공식]으로 `여신_base`, `여신_drift` 사용

2. `수신잔액.csv`
   `고객ID,기준일자,주차,상품구분,수신잔액`
   - 상품구분 = `수신_상품구분`, 값 = `수신_base`,`수신_drift`

3. `카드사용금액.csv`
   `고객ID,기준일자,주차,카드구분,사용금액,사용건수`
   - 카드구분 = `카드_구분`, 사용금액 = `카드_base`,`카드_drift`
   - 사용건수 = `int(round(사용금액 / 50000))`, 단 사용금액 0이면 0

4. `보험잔액.csv`
   `고객ID,기준일자,주차,상품구분,보험잔액`
   - 상품구분 = `보험_상품구분`, 값 = `보험_base`,`보험_drift`

## [값 공식] (모든 금액 공통)
```
week      = 이 에이전트가 맡은 주차번호 (1~12)
trend     = base * (1 + drift) ** (week - 1)
noise     = rng.normal(1.0, 0.03)  # rng = numpy default_rng(seed=week*1000003)
value     = int(round(trend * noise))
if base == 0: value = 0            # 미보유 고객은 항상 0 유지
value     = max(value, 0)          # 음수 금지
```
- `rng`는 **각 파일 시작 시 새로 생성**하고, 500명에 대해 순서대로 noise를 뽑는다.
- base/drift가 12주 내내 공유되므로 값이 주차에 따라 **완만한 추세**를 갖는다.

## 완료 전 자가검증 (반드시 통과)
각 CSV에 대해:
- 행 수 == 500
- 컬럼명/순서 == 위 명세와 정확히 일치
- 금액 컬럼 dtype 정수, 최소값 >= 0, NaN 없음
- `기준일자`가 calendar의 해당 주차 값과 일치
검증 실패 시 파일을 다시 생성한다.
