# IBK 내부 데이터 주간 대시보드 (AX 실습)

IBK 기업은행 **AX(AI Transformation) 실습** 데모. 내부 금융 데이터(여신·수신·카드·보험)를
**Python으로 분석/연산**하고 그 **집계 결과만** 웹 대시보드로 노출하여,
원본 개별 데이터의 **유출을 최대한 차단**하는 구조를 보여준다.

## 핵심 원칙 — 유출 방지

```
① 원본 데이터            ② 분석 계층 (Python)         ③ 표현 계층 (웹)
data/week_*/*.csv   ──►   analyze.py                ──►   index.html
(개별 잔액 = 민감)         통계·특이점 계산                집계 결과만 렌더
                          │                              (원본 CSV 미접근)
                          └─► output/*.json (집계값)
```

- 개별 고객의 잔액(민감)은 **분석 계층 Python 안에서만** 다뤄진다.
- 웹 페이지는 `output/*.json`(집계값)**만** 읽으며, 원본 CSV 경로에 접근하지 않는다.
- 따라서 raw 잔액이 브라우저로 새어나갈 통로 자체가 없다.

## 사용법

### 0. 요구사항
- Python 3.9+
- 패키지: `pandas`, `numpy`  (`pip install pandas numpy`)

### 1. 대시보드 실행 (가장 자주 쓰는 것)
데이터·분석 결과는 이미 생성되어 있으므로, 바로 웹 서버를 띄우면 된다.

```bash
# 프로젝트 루트에서
python3 -m http.server 8000
```
브라우저에서 **http://localhost:8000** 접속.

> ⚠️ `index.html`은 `fetch`로 JSON을 읽으므로 파일을 더블클릭(`file://`)하면 동작하지 않는다.
> 반드시 위처럼 **로컬 서버**로 열 것.

### 2. 분석 결과 갱신
원본 데이터가 바뀌었거나 분석 로직을 수정했을 때:

```bash
python3 analyze.py
```
→ `output/summary.json`, `output/anomalies.json` 재생성. 대시보드 새로고침 시 자동 반영.

### 3. 데이터 전체 재생성 (처음부터)
가상 데이터를 새로 만들려면 순서대로 실행:

```bash
python3 build_foundation.py      # ① 고객 풀·파라미터·달력·SPEC 생성
# ② 12개 주차 생성 (하나씩 실행하거나 반복문으로)
for i in $(seq -w 1 12); do python3 scripts/gen_week_$i.py; done
python3 inject_anomalies.py      # ③ 현실적 특이점 주입
python3 verify_all.py            # ④ 12주 무결성 교차검증 (PASS 확인)
python3 analyze.py               # ⑤ 집계 결과 생성
```

## 데이터 명세 (요약)

- **고객**: 500명, 12주 내내 동일 `고객ID`(예: `C100001`) 유지 → 주차 간 추세 분석 가능
- **주차**: 12주, 매주 금요일 스냅샷 (`2026-04-17` ~ `2026-07-03`)
- **금액**: 원(KRW) 단위 정수, 음수 없음
- **인코딩**: UTF-8 with BOM (`utf-8-sig`)
- 각 주차 파일은 정확히 **500행** (모든 고객 1회 등장, 미보유 시 0)

자세한 계약은 [`data/SPEC.md`](data/SPEC.md) 참고. 컬럼명·순서·타입을 지키지 않으면 분석 코드가 깨진다.

## 대시보드 구성

- **KPI 카드**: 4종 지표 최신주차 총액 + 전주대비 증감률
- **지표별 추이·집계**: 여신/수신/카드/보험 탭 전환 → 12주 총액 추이(라인), 지점별·연령대별·상품구분별 총액(막대)
- **특이점 분석**:
  | 유형 | 정의 | 의미 |
  |---|---|---|
  | ① 통계 이상치 | IQR 상단·3σ 초과 | 초고액 보유/사용 (VIP·리스크 집중) |
  | ② 급변 | 전주 대비 ±30% 이상 | 여신 급증(부실 신호), 수신 급감(이탈 신호) |
  | ③ 구조 신호 | 여신↑ & 수신↓ 동시 | 재무상태 악화 조기 경보 |
  | ④ 지속 추세 | 12주 단조 증가/감소 | 지속적 부채 증가 등 방향성 리스크 |

## 참고

- 생성 가상 데이터는 통계적으로 매끈해 특이점이 없어서, `inject_anomalies.py`로 지정 고객 33명에게만
  현실적 이상 신호를 주입한다. (구조 무결성은 `verify_all.py`로 유지 확인)
- 웹 페이지는 **고정 템플릿**이다. 매주 데이터만 갱신되며 레이아웃은 동일한 정형 리포트다.
  질문에 따라 화면이 달라지는 동적 방식은 향후 확장 과제(agent 실행 계층).
