# IBK 대출상품 상담챗봇 지식베이스 (demo3)

IBK기업은행 **개인사업자·소상공인 대출상품 8종**을 설명·추천하는 상담챗봇의 지식베이스(knowledge base)입니다. 벡터 DB 없이, **에이전트가 색인(`AGENTS.md`)을 보고 관련 파일만 읽어 답하는** 구조로 설계되었습니다.

> ⚠️ **데모용 자료입니다.** 금리·한도 등 실제 수치가 포함되어 있으나, 최신 정보는 IBK기업은행 공식 채널(홈페이지·영업점·고객센터)에서 확인해야 합니다.

## 설계 배경 (왜 이렇게 만들었나)

이 지식베이스는 다음 판단에 따라 설계되었습니다.

1. **코퍼스가 작다 → 벡터 RAG 대신 파일 기반.** 상품 8종 전체가 컨텍스트에 들어갈 만큼 작아, 임베딩·청킹·벡터DB는 과설계입니다. 표·자격조건이 잘리지 않고, 추천 시 전 상품을 함께 비교할 수 있습니다.

2. **"색인 + 온디맨드 파일 읽기"(agentic retrieval).** 저트래픽 서비스에서는 프롬프트 캐시가 항상 warm하지 않아 "매번 전체 주입"이 비쌀 수 있습니다. 그래서 `AGENTS.md`에는 **색인(라우팅 규칙)만** 두고, 실제 답변 시 관련 파일만 읽습니다.

3. **파일 단위는 "질문"이 아니라 "주제".** 질문마다 파일을 쪼개면 색인이 비대해지고 후속 질문이 조각나므로, 관련 질문 한 뭉치를 한 번에 답할 수 있는 **주제 단위**로 나눴습니다(`faq/` 9개).

4. **설명형 ↔ 추천형 분리.** 답이 안정적인 표준 질문은 `faq/`(설명형), 개인 상황에 따라 달라지는 추천은 `wiki/comparison.md`(추천형, 되묻기 기반)로 분리했습니다.

5. **원문이 진실, 정리본은 지도.** `faq/`·`wiki/`는 `loan_products_md/` 원문에서 파생된 정리본입니다. 정확한 수치의 최종 근거는 항상 원문이며, 금리 등은 **기준일과 함께** 안내합니다.

## 챗봇 동작 흐름

```
사용자 질문
  └─ [AGENTS.md 색인으로 라우팅]
       ├─ 표준 FAQ        → faq/<주제>.md 읽고 답 (출처·기준일 명시)
       ├─ 특정 상품 상세  → loan_products_md/<상품>.md
       ├─ 추천·개인화     → 되묻기 → wiki/comparison.md 로 후보 제시
       └─ 애매/광범위     → 원문 다수/전체 폴백 (코퍼스가 작아 안전)
  + 항상: 근거 없으면 영업점/고객센터 안내, 권유 금지, 심사 결과에 따름 고지
```

자세한 라우팅·답변 규칙은 [AGENTS.md](AGENTS.md)를 참고하세요.

## 데이터 출처 및 유지보수

- **출처 흐름**: `loan_products_raw/`(원본 png) → `loan_products_md/`(원문 텍스트) → `faq/`·`wiki/`(주제별 정리본).
