# IBK 상담 내용 요약 에이전트 (AX 실습)

IBK기업은행 영업점 직원의 **상담 내용을 유형에 맞게 요약**하고, **조회·동향 분석**까지 해 주는 에이전트 데모입니다.
별도 코드 없이, **Codex가 지시서(`AGENTS.md`)와 스킬(`SKILL.md`)을 읽고 에이전트처럼 동작**합니다.

---

## 무엇을 할 수 있나요?

세 가지 기능이 각각 **하나의 스킬**로 구현되어 있습니다.

| 스킬 | 설명 | 예시 명령 |
|------|------|-----------|
| 📋 **consult-list** (상담 내역 조회) | 어떤 상담이 있는지 파일 목록을 빠르게 보여줌 | "상담 내역 보여줘", "7월 1일 상담 목록" |
| 📝 **consult-summary** (상담 요약) | 특정 상담을 유형에 맞게 요약 → 문서 저장 | "7월 1일 두 번째 상담 요약해줘" |
| 📊 **consult-trend** (상담 동향 분석) | 기간을 정해 최근 공통 상담을 종합 리포트 | "지난 3일 상담 동향 알려줘" |

상담은 **기업대출 · 예적금 · 민원/사고** 세 유형으로 자동 분류되며, **유형마다 다른 항목**으로 요약됩니다.

## 폴더 구조

```
demo1/
├── AGENTS.md                        # 공통 지침 (항상 로드): 원칙·데이터규약·유형분류·지정해석
├── README.md                        # 이 문서
├── .agents/skills/                  # 기능별 스킬 (필요할 때만 로드)
│   ├── consult-list/SKILL.md        #   상담 내역 조회
│   ├── consult-summary/SKILL.md     #   상담 요약 (+ 유형별 템플릿)
│   └── consult-trend/SKILL.md       #   상담 동향 분석
├── data/                            # 상담 원문 70건 (테스트 입력)
│   ├── 20260701_01~10.txt
│   ├── ...
│   └── 20260707_01~10.txt
└── output/                          # 생성된 요약 문서 저장 위치 (*_summary.md)
```

- `data/`의 파일명은 `YYYYMMDD_NN.txt` 형식이며, **뒤 번호 `NN`은 그날의 상담 순번(시간 순)** 입니다.
- **2026-07-01 ~ 07-07** 7일간, 하루 10건씩 **총 70건**. 각 날짜에 세 유형이 섞여 있습니다.

### 왜 AGENTS.md와 스킬로 나눴나요?
- **AGENTS.md** = 세 기능이 공통으로 쓰는 "상식"(기본 원칙, 데이터 규약, 유형 분류) → 항상 로드
- **각 SKILL.md** = 그 기능만의 절차 + 언제 쓸지 알려주는 `description` → **관련 요청일 때만** 로드
- 덕분에 요청에 맞는 스킬만 가볍게 불러와 빠르고, 기능별로 따로 관리·재사용할 수 있습니다.

## 시작하기

이 폴더에서 Codex를 실행합니다.

```bash
cd demo1
codex
```

Codex는 `.agents/skills/`의 스킬을 자동으로 인식합니다. 아래 두 가지 방식으로 사용할 수 있어요.

- **그냥 자연어로 요청** → Codex가 `description`을 보고 알맞은 스킬을 **자동 선택**
  예) "상담 내역 보여줘" → `consult-list`
- **직접 지정** → `/skills` 로 목록을 보거나, 프롬프트에 `$consult-summary` 처럼 스킬을 **명시 호출**

## 사용법 & 예시

### 1) 상담 내역 조회 — `consult-list`
저장된 상담 **파일 목록**을 날짜별로 빠르게 보여 줍니다. (내용을 읽지 않아 즉시 응답)

```
상담 내역 보여줘
7월 1일 상담 목록 보여줘
```

출력 예시:

> 상담 내역은 총 70건입니다.
>
> ## 2026-07-01: 10건
> - 20260701_01.txt
> - 20260701_02.txt
> - …

### 2) 상담 요약 — `consult-summary`
특정 상담을 유형에 맞게 요약합니다. **파일명을 몰라도** 자연어로 지정할 수 있어요.

```
20260701_01.txt 요약해줘
7월 1일 두 번째 상담 요약해줘
7월 3일 마지막 상담 요약해줘
7월 1일 보이스피싱 상담 요약해줘
```

진행 순서:
1. 어떤 파일인지 확인 → 원문을 읽고 **상담 유형을 자동 판별**(근거 제시)
2. 유형에 맞는 **요약 방식 2~3가지를 추천하고 선택을 기다림**
   - 예: `상세 요약` / `핵심 3줄 요약` / `후속조치 중심 요약`
3. 선택한 방식으로 문서를 작성해 **`output/<파일명>_summary.md`** 로 저장

> 💡 추천 단계를 건너뛰려면 "바로 상세 요약으로 만들어줘"처럼 방식을 지정하면 됩니다.

### 3) 상담 동향 분석 — `consult-trend`
기간을 정해 여러 상담을 종합, **최근 공통적으로 발생하는 상담**을 리포트합니다.

```
최근 3일 상담 동향 알려줘
지난 일주일 어떤 상담이 많았어?
7월 1~2일 상담 종합해줘
전체 상담 동향 분석해줘
```

리포트에는 **유형별 분포 · 자주 발생한 세부 주제 Top N · 주의 신호 · 시사점**이 담깁니다.

> ℹ️ "최근"의 기준은 실제 오늘 날짜가 아니라 **`data/`에 있는 가장 최근 상담일(2026-07-07)** 입니다. 그래서 "최근 3일"은 07-05~07-07, "지난 일주일"은 07-01~07-07을 봅니다.

## 동작 원칙 (요약)

- 모든 응답·산출물은 **한국어**로 작성합니다.
- **원문에 없는 내용은 지어내지 않습니다.** 확인 안 되는 항목은 `정보 없음`으로 표기합니다.
- 금액·금리·이름 등 **구체 수치는 원문 그대로** 유지하고, 개인정보는 **마스킹**합니다.
- 동향 분석의 건수·비율은 **실제 파일 수를 세어** 산출합니다.

공통 규칙은 [`AGENTS.md`](AGENTS.md), 기능별 상세 절차는 각 [`SKILL.md`](.agents/skills)에 있습니다.

## 커스터마이징

- **상담 데이터 추가/교체**: `data/`에 `YYYYMMDD_NN.txt` 형식으로 넣으면 됩니다. (헤더: 상담일시·영업점·상담직원·상담채널 + 대화 본문)
- **요약 항목 변경**: `.agents/skills/consult-summary/SKILL.md`의 [유형별 요약 템플릿]을 수정하세요.
- **분류 유형 추가**: `AGENTS.md`의 [상담 유형 분류] 표에 유형과 판별 힌트를 추가하세요. (요약·동향 스킬이 함께 사용)
- **새 기능 추가**: `.agents/skills/`에 새 폴더와 `SKILL.md`(`name`·`description` 프론트매터 포함)를 만들면 됩니다.
