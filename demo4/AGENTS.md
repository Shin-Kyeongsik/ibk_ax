# AGENTS.md — 대출심사 정보 파악 시스템 (Codex 실행 지침)

이 저장소는 특정 기업의 **재무건전성·평판 리스크·주가 추이**를 수집·해석해
여신심사 참고용 HTML 리포트를 생성하는 도구다. 사용자가 "○○기업 대출심사 조사",
"여신 검토", "신용 리스크 파악" 등을 요청하면 **아래 워크플로우를 그대로 수행**한다.

## 핵심 원칙 (반드시 준수)
- **숫자·사실은 스크립트 출력에서만** 가져온다. 재무수치·주가·기사 내용을 지어내지 않는다.
  스크립트가 주지 않은 값은 "확인 불가"로 표기한다.
- 수집·계산은 `scripts/`의 파이썬 CLI가 **결정적으로** 처리한다. 너는 그 결과를 **해석**하고 종합한다.
- 이 리포트는 **의사결정 보조 자료**다. "대출 승인/거절"을 단정하지 말고, 리스크 신호와 근거를 정리한다.
- 상충하는 신호(예: 재무 양호 + 최근 소송)는 감추지 말고 함께 제시한다.

---

## 디렉터리 규칙 (반드시 준수)
결과물을 깔끔하게 유지하기 위해 파일 종류별 위치를 고정한다.

| 종류 | 위치 | 비고 |
|---|---|---|
| **파이썬 스크립트(.py)** | `scripts/` | 수집·계산·렌더링 코드는 모두 여기. 실행은 `python3 scripts/<파일>.py` |
| **JSON(payload·중간 수집)** | `meta/` | payload 및 스크립트 출력 JSON은 모두 여기에 저장. 없으면 먼저 생성 |
| **최종 HTML 리포트** | 저장소 루트 | 루트에는 결과 HTML만 남긴다 |

- 스크립트는 **반드시 `scripts/` 경로로** 실행한다(`python3 scripts/dart_client.py ...`).
  파이썬이 스크립트 디렉터리를 자동으로 import 경로에 넣으므로, 저장소 루트에서 그대로 실행하면 된다.
- 새 파이썬 파일을 만들 일이 있으면 **`scripts/` 안에** 만든다. 새 JSON은 **`meta/` 안에** 만든다.

## 워크플로우

### 0. 사전 확인
- 조사 대상 **기업명**을 확정한다. 모호하면 사용자에게 한 번 되묻는다.
- DART 인증키는 `scripts/dart_client.py` 에 **하드코딩**돼 있어 별도 설정이 필요 없다(데모용).
  (환경변수 `DART_API_KEY` 를 설정하면 그 값이 우선한다.)

### 1. 대상 기업 확정 + 상장 여부 판별
```bash
python3 scripts/dart_client.py "<기업명>" --pretty
```
- 출력의 `선택된_기업` 이 의도한 기업인지 확인한다. `기타_후보` 가 있고 모호하면 사용자에게 확인한다.
- `기본정보.상장주식코드`(stock_code) 유무로 **상장/비상장** 을 판별한다.
  이 종목코드는 3단계 주가 수집에 사용한다.

### 2. 세 도메인 수집·해석
아래 3개 스크립트를 실행하고, 각 출력을 **해당 도메인 기준(아래 '도메인별 해석')으로 해석**한다.
스크립트는 독립적이라 순서는 무관하다.

```bash
# (1) 재무·기본정보  — DART 키는 스크립트에 내장(데모)
python3 scripts/dart_client.py "<기업명>" --year <연도> --pretty

# (2) 평판·뉴스 리스크 — 키 불필요
python3 scripts/news_search.py "<기업명>" --days 180 --limit 30 --pretty

# (3) 주가 추이 — 상장사만, 1단계의 종목코드 사용
python3 scripts/stock.py <종목코드> --days 365 --series --pretty
```
- 비상장이면 (3)은 생략하고 payload의 `stock` 을 `{"not_listed": true}` 로 둔다.
- (1)의 `재무정보` 가 비어 있으면 `--year` 를 1~2년 낮춰 재시도한다(사업보고서 제출 시차).

### 3. payload(JSON) 조립
해석 결과를 아래 스키마의 JSON으로 만들어 **`meta/` 디렉터리 안에** 저장한다(예: `meta/payload.json`).
중간 수집 결과(스크립트 출력 JSON)를 파일로 남길 때도 **모두 `meta/` 아래에** 둔다.
저장소 루트에는 최종 HTML 리포트만 남겨 결과물을 깔끔하게 볼 수 있도록 한다. (`meta/` 가 없으면 먼저 생성)
- **숫자·사실 필드**(`ratios`, `accounts`, `metrics`, `series`, 기사 `title/source/date`)는 스크립트 출력값을 **그대로** 넣는다.
- **해석 필드**(`comment`, `assessment`, `overall.summary`)는 네 판단을 요약해 넣는다.
- **`tone`** 은 리포트 색상을 결정한다(라벨과 분리): `good`=양호/저위험, `warn`=주의, `bad`=위험, `none`=해당없음.

```json
{
  "company":  {"name","ceo","industry","established","listing","biz_no","address"},
  "financial":{"fiscal_year","fs_type","ratios":{...},"accounts":{...},
               "assessment":{"level","tone"},"comment"},
  "reputation":{"period","reviewed_count",
                "risks":[{"category","tone","title","source","date","summary"}],
                "assessment":{"level","tone"},"comment"},
  "stock":    {"code","period","metrics":{...},"series":[{"date","close"}],
               "assessment":{"level","tone"},"comment"},   // 상장사만
  "overall":  {"level","tone","summary"}
}
```
(필드 상세는 `scripts/render_report.py` 상단 주석 참조)

### 4. HTML 리포트 생성
```bash
python3 scripts/render_report.py meta/payload.json -o "<회사명>_대출심사리포트.html"
```
- 생성된 HTML 경로를 사용자에게 안내한다. **HTML을 직접 작성하지 말 것** — 렌더러가 고정 템플릿에 값만 채운다.
- 사용자가 요약만 원하면 마크다운으로도 함께 제시한다.

## 도메인별 해석 기준

### 재무 (dart_client.py 결과)
스크립트가 계산한 비율을 **여신 관점**으로 해석한다:
- **부채비율**: 100% 이하 우량 / 200% 초과 주의 / 400% 초과 위험 (업종별 상대성 감안)
- **유동비율**: 100% 미만이면 단기 지급능력 우려
- **자기자본비율**: 낮을수록 재무 레버리지 과다
- **수익성 추세**: 영업이익률·순이익률의 방향 (적자전환·이익 급감은 경고)
- 종합 등급: **양호(good) / 보통(warn) / 미흡(bad)**

### 평판 (news_search.py 결과)
스크립트는 키워드로 1차 태깅만 한다. **최종 선별은 네 몫이다**:
- 맥락을 보라: "부도 위기 극복", "소송 승소" 처럼 오히려 긍정이거나 해당 기업과 무관한 기사는 제외.
- 중대도 순위: 부도·파산·회생·자본잠식·상장폐지 > 횡령·배임·분식회계 > 소송·제재·과징금 > 실적악화·구조조정 > 오너 리스크.
- 부정 이슈가 정말 없으면 "특이 리스크 없음"이라 분명히 말한다(억지로 만들지 말 것).
- 각 이슈에 근거 기사 제목·언론사·날짜를 붙인다.
- 종합 등급: **낮음(good) / 보통(warn) / 높음(bad)**

### 주가 (stock.py 결과, 상장사만)
시장의 기업 평가로 해석하되 **시장 전반 영향과 구분**한다:
- **기간등락률**: 장기 하락은 실적·전망 악화 신호일 수 있음
- **최고점 대비 낙폭**: 큰 낙폭(-50%+)은 시장 신뢰 훼손 신호
- **연율화변동성**: 높을수록(60%+) 불확실성 큼
- 주가 하락이 시장 전체 하락 때문일 수 있으므로 단정하지 말고 "가능성"으로 서술. **보조 참고지표**임을 명시.
- 종합 등급: **낮음(good) / 보통(warn) / 높음(bad)**
