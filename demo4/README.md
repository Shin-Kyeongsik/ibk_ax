# 대출심사 정보 파악 시스템 — IBK AX 실습 (Codex)

특정 기업에 대해 **재무건전성 · 평판 리스크 · 주가 추이**를 자동 수집·해석해
여신심사 참고용 HTML 리포트를 생성하는 데모. **OpenAI Codex CLI**에서 실행한다.

> ⚠️ 공개 데이터(DART·뉴스·시세) 기반 **의사결정 보조 도구**입니다.
> 실제 여신 승인/거절 판단을 대신하지 않으며, 공식 신용등급·내부 여신정보는 포함하지 않습니다.

---

## 1. 아키텍처 한눈에 보기

Codex는 저장소 루트의 **`AGENTS.md`** 를 지침으로 읽어, 단일 에이전트가 아래 워크플로우를 수행한다.

```
사용자: "○○기업 대출심사 정보 조사해줘"
   │
   ▼
[Codex 에이전트]  ── AGENTS.md 지침에 따라 진행
   │  1) 대상 기업 확정 · 상장 여부 판별   (dart_client.py)
   │  2) 세 도메인 수집·해석
   │       ├─ 재무·기본정보  → dart_client.py
   │       ├─ 평판 리스크     → news_search.py
   │       └─ 주가 추이       → stock.py   (상장사만)
   │  3) 결과 종합 → payload(JSON)   (meta/ 에 저장)
   │  4) render_report.py → HTML 리포트  (루트에 저장)
   ▼
대출심사 요약 리포트 (HTML)
```

**설계 원칙**: 수집·계산은 파이썬 스크립트가 **결정적**으로, 도메인 해석·종합은 **에이전트**가 담당.
숫자·사실은 스크립트 출력에서만 오므로 환각(hallucination)을 차단한다.

### 파일 구성
| 경로 | 역할 |
|---|---|
| `AGENTS.md`               | **Codex 진입점** — 전체 워크플로우·해석 기준·가드레일 |
| `scripts/dart_client.py`  | DART에서 기업개황·재무제표 수집 + 재무비율 계산 |
| `scripts/news_search.py`  | Google News에서 뉴스 수집 + 여신 리스크 키워드 태깅 |
| `scripts/stock.py`        | 네이버 금융에서 주가 수집 + 등락률·변동성·낙폭 계산 |
| `scripts/render_report.py`| 종합 JSON을 고정 템플릿 HTML 리포트로 렌더링 |
| `meta/`                   | 생성된 payload·중간 수집 JSON 저장소 (루트는 최종 HTML만 유지) |

> Codex에는 Claude Code의 스킬/sub-agent 스폰 메커니즘이 없으므로, 오케스트레이션은
> `AGENTS.md`에 통합해 **단일 에이전트가 스크립트를 순차 실행**하는 방식으로 동작한다.

## 2. 사전 준비 (실습 시작 전)

### 2-1. 환경
```bash
python3 --version          # 3.10 이상 권장
pip install -r requirements.txt   # 외부 의존성은 requests 하나뿐
```

### 2-2. DART 인증키 (데모용 하드코딩됨 — 별도 설정 불필요)
이 데모는 DART 인증키를 `scripts/dart_client.py` 상단 `DEFAULT_API_KEY` 에 **하드코딩**해 두었다.
바로 실행하면 된다. 자신의 키로 바꾸려면 그 값을 교체하거나, 환경변수로 덮어쓴다:
```bash
export DART_API_KEY=발급받은_40자리_키   # (선택) 설정 시 하드코딩 값보다 우선
```
- 키 발급(무료): https://opendart.fss.or.kr/ → **인증키 신청/관리 → 인증키 신청**
- ⚠️ 실제 운영/공개 배포 시에는 키를 코드에서 분리(환경변수·시크릿 매니저)할 것.

## 3. 실행 방법

### 3-1. 개별 스크립트 단독 실행 (스크립트 동작 확인용)
```bash
# 재무·기본정보 (DART 키는 스크립트에 내장)
python3 scripts/dart_client.py "삼성전자" --year 2023 --pretty

# 평판 리스크 (키 불필요)
python3 scripts/news_search.py "삼성전자" --days 180 --risk-only --pretty

# 주가 추이 (종목코드는 키 불필요)
python3 scripts/stock.py 005930 --days 365 --pretty
```

### 3-2. 전체 파이프라인 (Codex)
저장소 루트에서 Codex를 실행하고 자연어로 요청한다(DART 키는 이미 내장):
```bash
codex          # 저장소 루트에서 실행 → AGENTS.md 자동 로드
```
Codex 프롬프트에 입력:
```
삼성전자 대출심사 정보 조사해줘
```
→ Codex가 `AGENTS.md` 워크플로우에 따라 3개 스크립트를 실행·해석하고,
   종합 후 `<회사명>_대출심사리포트.html` 을 생성한다. 브라우저로 열어 확인.

> Codex가 스크립트 실행(셸 명령)을 승인 요청할 수 있다. 실습에서는 승인하거나
> 워크스페이스 쓰기 권한을 미리 허용해두면 흐름이 매끄럽다.

## 4. 실습 시연 시나리오

교육 효과를 위해 **성격이 다른 3개 기업**을 준비했다.

| 시나리오 | 예시 기업 | 보여주는 포인트 |
|---|---|---|
| ① 우량 상장사 | **삼성전자** | 낮은 부채비율·높은 유동비율, 리스크 없음 → "양호" 리포트 |
| ② 리스크 기업 | **홈플러스** | 뉴스에서 파산·회생 등 부정 이슈 다수 태깅 → "리스크 높음" |
| ③ 고변동 코스닥 | **에코프로비엠** | 큰 주가 낙폭·높은 변동성 → 주가 리스크 부각 |

각 사례로 "재무는 좋은데 평판이 나쁜 경우", "상장/비상장 차이" 등을 대비해 보여주면 좋다.

## 6. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| DART `[020] 사용한도 초과` | DART 일일 호출 한도 초과 → 잠시 후 재시도 |
| DART `[010/011] 등록되지 않은/사용중지 키` | `dart_client.py` 의 `DEFAULT_API_KEY` 를 유효한 키로 교체 |
| 재무정보가 비어 있음 | 해당 연도 사업보고서 미제출 → `--year` 를 1~2년 낮춰 재시도 |
| 뉴스가 안 나옴 | 기업명이 너무 일반적이거나 최근 기사 부재 → `--days` 늘리기 |
| 주가 `시세 조회 결과 없음` | 비상장이거나 종목코드 오류 → 6자리 코드 확인 |
| 기업명이 여러 개로 잡힘 | `기타_후보` 확인 후 정확한 상호명 사용 |
